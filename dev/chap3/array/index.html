Array initialization and indexing

Initializing an array in Julia is simple. You can initialize an array with different types of elements.

julia> zero_vector = zeros(3) # zero vector3-element Vector{Float64}:
 0.0
 0.0
 0.0
julia> direct_matrix = [5 2 1; 1 4 5; 2 4 9] # explicit3×3 Matrix{Int64}:
 5  2  1
 1  4  5
 2  4  9
julia> rand_vector = randn(Float32, 3, 3) # random normal distribution3×3 Matrix{Float32}:
 -1.5838     0.011126  -0.150038
 -0.417944   0.959746  -1.43772
  0.882631  -0.128281   1.73865
julia> step_vector = collect(1:3)  # collect from a range3-element Vector{Int64}:
 1
 2
 3
julia> uninitialized_vector = Vector{Int}(undef, 3) # uninitialized vector of size 33-element Vector{Int64}:
 140524004156336
 140524076446528
 140524225059744

  • Julia array is fast. In memory, it is a contiguous block if the element type is bitstype - having a fixed size in memory.

    julia> isbitstype(Int)true
    julia> isbitstype(Complex{Float64})   # complex number has a fixed sizetrue
    julia> sizeof(Complex{Float64})16
    julia> isbitstype(typeof(('a', 2, false)))  # tuple has a fixed sizetrue
    julia> sizeof(typeof(('a', 2, false)))24
    julia> isbitstype(typeof(randn(3)))   # array does not have a fixed sizefalse
    julia> sizeof(typeof(randn(3)))  # an array of array may cause slow indexing.ERROR: Type Array does not have a definite size.

  • Julia array memory layout is column-major. Looping over the first index is faster than looping over the last index.

    julia> function mysum_col(A::AbstractMatrix)
           s = zero(eltype(A)) # zero of the element type of A
           for j in 1:size(A, 2) # loop over the second index
               for i in 1:size(A, 1)  # loop over the first index
                   s += A[i, j]
               end
           end
           return s
       endmysum_col (generic function with 1 method)
    julia> function mysum_row(A::AbstractMatrix)
           s = zero(eltype(A)) # zero of the element type of A
           for i in 1:size(A, 1)  # loop over the first index
              for j in 1:size(A, 2) # loop over the second index
                   s += A[i, j]
               end
           end
           return s
       endmysum_row (generic function with 1 method)

    For small scale matrix, their performance is similar since the matrix can be cached well. However, for large scale matrix, the column-major layout is much faster due to the cache locality.

    julia> using BenchmarkTools

julia> A = rand(10000, 10000);

julia> @btime mysum_col($A)
  85.885 ms (0 allocations: 0 bytes)
4.999830721534851e7

julia> @btime mysum_row($A)
  189.794 ms (0 allocations: 0 bytes)
4.9998307215344414e7

  • Julia array indexing starts from 1. It is different from C, Python, and R, which start from 0. 😞

    julia> A = [1, 2, 3]3-element Vector{Int64}:
 1
 2
 3
    julia> A[1]     # the first element1
    julia> A[end]   # the last element3
    julia> A[1:2]   # the first two elements2-element Vector{Int64}:
 1
 2
    julia> A[2:-1:1] # the first two elements in the reversed order2-element Vector{Int64}:
 2
 1
    julia> B = [1 2 3; 4 5 6; 7 8 9];
    julia> B[1:2]   # the first two elements, returns B[1,1] and B[2,1] since B is column-major2-element Vector{Int64}:
 1
 4
    julia> B[1:2, 1:2] # returns a submatrix2×2 Matrix{Int64}:
 1  2
 4  5

Broadcasting

Julia has a powerful broadcasting mechanism. It is a way to apply a function to each element of an array. The broadcasting is done by adding a dot . before the function name.

julia> x = 0:0.1:2π0.0:0.1:6.2
julia> y = sin.(x) .+ cos.(3 .* x);
using Plots; gr(dpi=100)  # for high-quality plots, we suggest using `Makie.jl`
Plots.plot(x, y; label="sin(x) + cos(3x)")

Loop fusion

The broadcasting is very efficient due to loop fusion. In the above example, only one loop is needed to calculate sin and cos for each element of x and only one array is allocated to store the result.

When shapes do not match

julia> A = [1 2 3; 4 5 6]2×3 Matrix{Int64}:
 1  2  3
 4  5  6
julia> B = [1 2 3]1×3 Matrix{Int64}:
 1  2  3
julia> A .+ B   # B is broadcasted to match the size of A2×3 Matrix{Int64}:
 2  4  6
 5  7  9

B is a (column) vector, which means its shape is (3, 1). The broadcasting is done by repeating the vector to match the shape of A.

Unwanted broadcasting

Consider we have a vector, and we want to scale it with factors 1, 2, 3 and store the results in to a vector, i.e. we will get a vector of vectors. The following code will not work as expected.

julia> [3,2,1,0] .* (1:3)ERROR: DimensionMismatch: arrays could not be broadcast to a common size; got a dimension with lengths 4 and 3

Note a range is an iterable object. The broadcasting operation works on any iterable object, hence it tries to broadcast over the elements of the vector. A shape mismatch error is raised, which is unwanted.

In this case, we can use Ref to protect the vector from broadcasting.

julia> Ref([3,2,1,0]) .* (1:3)3-element Vector{Vector{Int64}}:
 [3, 2, 1, 0]
 [6, 4, 2, 0]
 [9, 6, 3, 0]

We can see the vector is treated as a whole.

Array performance tips

  1. Fix the type of an array

Arrays with a fixed type are faster than arrays with abstract types. Any type vector is the most flexible, but it is also very slow.

julia> vany = Any[]  # same as vany = []Any[]
julia> typeof(vany)Vector{Any} (alias for Array{Any, 1})
julia> push!(vany, "a")1-element Vector{Any}:
 "a"
julia> push!(vany, 1)2-element Vector{Any}:
  "a"
 1

Fixed typed vector is more restrictive.

julia> vfloat64 = Float64[]Float64[]
julia> vfloat64 |> typeofVector{Float64} (alias for Array{Float64, 1})
julia> push!(vfloat64, "a")ERROR: MethodError: Cannot `convert` an object of type String to an object of type Float64

Closest candidates are:
